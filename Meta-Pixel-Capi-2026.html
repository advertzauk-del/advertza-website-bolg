What Is the Meta Pixel: and Why It Broke

The Meta Pixel is a JavaScript snippet installed on your website. When a visitor lands on a page, the pixel fires from their browser and sends event data back to Meta. The pixel works through the user's browser: meaning it is entirely dependent on the browser environment allowing it to fire. This worked reliably until April 2021.

The iOS 14.5 Problem: Five Years Later

When Apple released iOS 14.5 in April 2021, the majority of iPhone users opted out of cross-app tracking. For Meta, this meant their pixel became blind to conversions from opted-out iOS users: a group that represents 30–60% of traffic in the UK. In accounts Advertza manages, pixel-only setups in 2026 consistently show event match quality scores below 6.0 and under-reported purchase volumes of 40–55% compared to actual platform revenue. The pixel alone is not viable infrastructure.

What Is CAPI and Why You Need Both

CAPI sends your website's conversion events directly from your server to Meta's API. Instead of the user's browser sending event data (which iOS can block), your web server sends the same data independently of what the user's device allows. The server-to-server connection is not subject to browser privacy settings, iOS app tracking permissions, ad blockers, or third-party cookie deprecation.

Deduplication Is Not Optional

When both pixel and CAPI fire for the same purchase, Meta receives two event reports for one conversion. Without deduplication, Meta counts both: inflating reported conversions and feeding corrupted signal to the algorithm. Deduplication works by attaching a unique event_id to each event. The pixel sends it in the browser. CAPI sends the same event_id from the server. Meta matches them and counts the pair as one conversion.

For Purchase events: use the order ID as the event_id. For Lead events: use a combination of timestamp and a random suffix. For ViewContent: generate a UUID client-side.

GTM Setup: Step-by-Step

Google Tag Manager is the recommended installation method for all websites not running a direct platform integration like Shopify. GTM separates pixel management from your website codebase: you can add, modify, or troubleshoot tags without touching your site's HTML.

01

Create Your Pixel in Meta Events Manager

Open Meta Business Suite → Events Manager → Connect Data Sources → Web. Select 'Meta Pixel', give it a clear name, and copy the Pixel ID from the confirmation screen.

02

Install the Base Code Tag in GTM

In GTM: Tags → New → Custom HTML. Paste the pixel base code. Trigger: All Pages. Publish the container. Verify with Meta Pixel Helper Chrome extension: every page should show 'Active'.

03

Add Standard Event Tags

Create individual Custom HTML tags for each standard event. Purchase tag must include value and currency parameters: without these, Meta cannot optimise for ROAS. Trigger Purchase on the order confirmation page URL.

04

Add event_id to Every Event Tag

Before publishing, add eventID to each fbq('track',...) call. For Purchase, use the order ID. Store this ID in a dataLayer variable: pass the exact same value to CAPI in the next phase.

CAPI Gateway Setup (No Code Required)

CAPI Gateway is Meta's hosted server-side solution: you connect your website to Meta's infrastructure without deploying any server-side code yourself. For most advertisers, this is the fastest and most reliable path to a working CAPI setup.

01

Access CAPI Setup in Events Manager

In Events Manager, find your pixel → Add Events → Using Conversions API → CAPI Gateway. The guided wizard opens. Choose 'Set up a new gateway'. Meta provisions your gateway endpoint.

02

Install the Gateway Connection Plugin

Meta provides a lightweight script or WordPress plugin to connect your site to the gateway endpoint. For Hostinger-hosted sites, upload the script via cPanel File Manager to the root directory.

03

Map Events and Enable Deduplication

In the gateway wizard, map each server event to the corresponding pixel event. Purchase → Purchase. Confirm the event_id field is mapped: this triggers Meta's deduplication logic.

04

Verify in Test Events Tool

Navigate to Events Manager → Test Events. Complete a test action. You should see paired events: one labelled 'Browser' (pixel) and one 'Server' (CAPI): with a 'Deduplicated' status indicator.

Shopify CAPI: 2-Minute Setup

Shopify CAPI: 2 Steps

Step 1: In Shopify Admin, go to Apps → Facebook and Instagram (install from Shopify App Store if not present). Navigate to Settings → Data Sharing → Conversions API. Toggle server-side events to On.

Step 2: Set sharing level to Maximum. Shopify handles event mapping, deduplication via order ID, and all data transmission automatically.

Cross-client data from Advertza shows Shopify CAPI at Maximum sharing level consistently achieves event match quality scores of 8.0–9.0 out of 10: among the best across any implementation method. One caveat: if you previously installed the Meta Pixel manually via GTM, disable it before activating the Shopify Facebook app's pixel. Running both fires duplicate uncorrelated events that confuse deduplication logic.

What is the Meta Pixel and do I still need it in 2026?+
The Meta Pixel is a JavaScript snippet that fires from a user's browser tracking page views, product views, add-to-cart events, and purchases. In 2026, the pixel alone is no longer sufficient: iOS 14.5 blocks browser-based tracking for opted-out users, meaning pixel-only accounts miss 50%+ of actual conversions. You still need the pixel, but you need the Conversions API running alongside it. Running pixel alone in 2026 means making every campaign decision on approximately half your actual conversion data.
What is CAPI and how does it work?+
CAPI (Conversions API) sends your conversion events directly from your server to Meta, bypassing the user's browser entirely. This server-to-server connection is not affected by iOS tracking restrictions, ad blockers, or cookie deprecation. Combined with the pixel and proper event deduplication, dual tracking recovers close to 100% of actual conversion attribution compared to the approximately 50% that pixel-only accounts receive.
How do I set up CAPI without coding?+
Meta's CAPI Gateway option requires zero code. In Events Manager, navigate to your pixel → Add Events → Conversions API → CAPI Gateway. For Shopify: install the Facebook and Instagram app, go to Settings → Data Sharing → Conversions API, enable server-side events set to Maximum. This takes approximately two minutes and requires no technical knowledge.
What is event deduplication and why does it matter?+
When both pixel and CAPI are active, every conversion sends two events to Meta. Without deduplication, Meta counts both: inflating reported conversions and corrupting the algorithm's optimisation signal. Deduplication works by adding a unique event_id to every event: the same ID in both the pixel and CAPI event. For Purchase events, use the order ID as the event_id. It is already unique per transaction.
How do I test if my tracking is working correctly?+
Use two tools: Meta Pixel Helper (Chrome extension) for browser-side verification, and the Test Events tool inside Events Manager for end-to-end verification. A correctly configured dual setup shows paired browser and server events with 'Deduplicated' status. Check your Event Match Quality score: scores of 8.0+ indicate excellent parameter coverage. Scores below 6.0 mean significant attribution gaps.
